# AI Ninja Program — Syllabus

> A selective training program for security architects and engineers across the Americas.

---

## Program Overview

The AI Ninja Program is a 15-week cohort-based training program that takes security professionals from zero AI/ML knowledge to building AI-powered security tools and positioning Check Point's AI Security products with technical depth. Graduates earn tiered certification and walk away with a portable customer demo they can use in sales engagements immediately.

This is not a passive course. Every week includes hands-on coding, every stage ends with an assessment gate, and the program culminates in a live capstone presentation to a review panel.

**Duration:** 15 weeks
**Format:** Self-paced modules + weekly live sessions
**Cohort size:** 12–16 participants
**Regions:** Americas (NA, LATAM, Brazil)

---

## Program Objectives

By the end of this program, participants will be able to:

1. **Explain** AI/ML concepts to customers and internal stakeholders without jargon
2. **Evaluate** vendor AI claims with technical depth ("What algorithm? What training data? What's the false positive rate?")
3. **Build** working ML classifiers for security use cases (phishing detection, intrusion detection, anomaly scoring)
4. **Design** and critique neural network architectures for security applications
5. **Demonstrate** a RAG-based security assistant live to a customer audience
6. **Position** Check Point's AI Security products (Workforce AI Security, AI Agent Security, AI Guardrails) with technical depth
7. **Lead** AI-driven technical discussions in sales engagements and architecture reviews

---

## Who This Is For

This program is designed for:

- **Security Architects** who evaluate and design security solutions for customers
- **Security Engineers** who build, deploy, and operate security controls
- **Pre-sales Engineers** who need to articulate AI capabilities in customer-facing engagements

### Prerequisites

| Requirement | Details |
|-------------|---------|
| Python proficiency | Comfortable with variables, loops, functions, classes, list comprehensions |
| Security domain knowledge | Working knowledge of firewalls, SIEM, endpoint detection, network security |
| Time commitment | 5-8 hours per week for 15 weeks |

### Not Required

- Prior ML, statistics, or data science experience
- Mathematics beyond basic algebra
- Any specific security certification
- GPU or cloud compute resources

## Three-Tier Certification

The program awards progressive certifications. Each tier builds on the previous one.

| Tier | Name | Weeks | Capability |
|------|------|-------|------------|
| **3** | **AI Ninja** | 11–13 | Can build AI-powered security tools and lead AI-driven engagements |
| **2** | **AI Practitioner** | 8–10 | Can prototype a neural network and assess AI solution architectures |
| **1** | **AI Foundations** | 1–7 | Can explain AI/ML to a customer and evaluate vendor AI claims |

> Each tier builds on the previous one — Tier 1 is required for Tier 2, and Tier 2 is required for Tier 3.

| Tier | Stages Completed | Assessment Gates | Credential |
|------|-----------------|------------------|------------|
| **Tier 1 — AI Foundations** | Stages 0, 1, 2 | Gates 1 + 2 | AI Foundations Certified |
| **Tier 2 — AI Practitioner** | + Stage 3 | + Gate 3 | AI Practitioner Certified |
| **Tier 3 — AI Ninja** | + Stages 4, 5 | + Gate 4 (live demo) | AI Ninja |

### Week 15 (end) — Gate 4: Capstone Demo Day (Live)

Each participant presents their Security Analyst Assistant AND Check Point AI Security positioning to a review panel.

| Component | Details |
|-----------|---------|
| **Duration** | 7-minute demo + 5-minute Q&A per participant |
| **Audience** | Tech lead + 2-3 stakeholders (engineering managers, sales leaders) |
| **Demo content** | Part 1: RAG assistant grounded in retrieved documents. Part 2: Check Point AI Security product positioning (choose one: Workforce dashboard, Guardrails attack demo, or MCP workflow). |
| **Evaluation** | Rubric scoring (see assessments folder) |

#### Capstone Rubric

| Criterion | 1 (Needs Work) | 3 (Meets Standard) | 5 (Exceptional) |
|-----------|----------------|--------------------|--------------------|
| **Technical depth** | Cannot explain how RAG works | Explains chunking, embedding, retrieval, generation | Discusses trade-offs (chunk size, model choice, retrieval accuracy) |
| **Demo quality** | Demo fails or produces irrelevant answers | Demo runs, answers are grounded in documents | Pre-built queries showcase impressive, accurate results |
| **Customer readiness** | Reads from notes, uses jargon | Clear narrative: problem → solution → demo → value | Handles live questions confidently, ties to business outcomes |
| **Code quality** | Solution is copied from template | Customised with own documents or features | Extended with additional capabilities (multi-doc, follow-up questions) |

**Passing score:** Average of 3.0 or higher across all criteria.

**Pass → Tier 3: AI Ninja**

## Participant Expectations

### Before the Program

- [ ] Python 3.10+ installed and working
- [ ] Repository cloned and virtual environment set up
- [ ] Stage 1 dependencies installed (`pip install numpy pandas matplotlib scikit-learn`)
- [ ] Confirm weekly availability for live sessions

### During the Program

- **Complete exercises before each live session** — sessions are for discussion, not first exposure
- **Post weekly in the cohort channel** — what you learned, what confused you, or what clicked
- **Review your pair partner's capstone code** in Stages 2 and 4
- **Attend live sessions** — or watch the recording within 48 hours
- **Attempt assessments on first pass** — retakes are allowed but first-attempt pass rate is tracked

### Falling Behind

If you fall more than one week behind:
1. Notify the program lead
2. Your pair partner and facilitator will check in
3. You can complete a "fast track" version of missed exercises (capstone-only, skip individual exercises)
4. Assessment gates can be rescheduled once per participant

## Facilitator Information

Each cohort is led by a facilitator (the program tech lead or a Tier 3 graduate from a previous cohort).

### Facilitator Responsibilities

- Run weekly live Q&A sessions (60 min each)
- Proctor assessment gates
- Monitor cohort channel activity and follow up with quiet participants
- Review and score Gate 4 capstone demos (with 2-3 co-reviewers)
- Provide the final tier certification for each participant

### Facilitator Guides

A standardised facilitator guide is provided for each lecture. See `facilitator_guides/` for the template and per-lesson notes including timing, discussion prompts, common questions, and key points to emphasise.
