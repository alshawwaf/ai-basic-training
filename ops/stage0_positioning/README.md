# Stage 0 — AI for Security Positioning

## Goal
Build the vocabulary, market awareness, and conversational skill to discuss AI in cybersecurity with customers, partners, and internal stakeholders — before writing a single line of code.

---

## Why This Stage Comes First

Stages 1-4 teach you **how AI works**. Stage 0 teaches you **how to talk about AI** in the context of security sales and architecture engagements.

An architect who understands k-means clustering is useful. An architect who can say "their anomaly detection is k-means with a 30-day sliding window — here's why our approach handles concept drift better" wins deals.

## Prerequisites

- None — this stage requires no coding or ML background
- Complete this stage before starting Stage 1

## After This Stage

You will be able to:
- Explain what AI/ML means in cybersecurity without using jargon
- Identify what competitors are actually doing under the hood
- Handle common objections ("Isn't this just pattern matching?")
- Ask discovery questions that uncover AI opportunities in customer environments
