LLMs vs classic ML

You've now built intuition for tokens, embeddings, and attention. Here's when to use LLMs vs the models from Stages 1-3.

📈
Classic ML
Stages 1-3
Tabular data

Text / unstructured
Classic ML (Stages 1-3)LLMs (Stage 4)
TrainingTrain from scratch on your dataPre-trained on trillions of tokens
Data neededHundreds to thousands of labelled samplesZero-shot or few-shot (no labels!)
InputStructured features (numbers)Raw text
ParametersHundreds to thousandsBillions
InferenceMicrosecondsSeconds
CostFree (runs locally)Pay per token (API)
ExplainabilityFeature importance, decision pathsBlack box

What you learned

StepConceptKey takeaway
0Next-token predictionLLMs predict one token at a time; everything else emerges
1TokenisationText → token IDs using a vocabulary
2Vocabulary limitsSmall vocab = lost meaning; BPE solves this
3EmbeddingsEach token becomes a dense vector capturing semantics
4Cosine similaritySimilar meanings → similar vectors → measurable distance
5AttentionEach word selectively focuses on relevant context
6Context vectorsAttention creates context-aware representations

Use LLMs for:

  • Parsing unstructured threat reports
  • Summarising incident logs
  • Semantic search over threat intel
  • Generating detection rules from descriptions
  • Chatbot interfaces for SOC analysts

Use classic ML for:

  • Classifying network connections (attack/benign)
  • Anomaly detection on structured logs
  • Malware feature analysis
  • User behaviour analytics (UEBA)
  • Alert triage and prioritisation
